# Kernel Execution Contract v7.3 ‚Äì Formal Specification

## 1. Execution Axioms (v7.2 ‚Üí v7.4)

1. **Deterministic Execution Invariant** ‚Äì For any intent `I` with seed `S`, the ECVM produces a unique execution trace `T` such that:
   ```
   stableExecutionHash({ intent: I, ecvm_seed: S, scheduler_trace: T.scheduler, io_trace: T.io }) = H
   ```
   The hash `H` is immutable across all replicas.
2. **Lease Uniqueness** ‚Äì At any moment there is at most one `kernel_kebl_leases` row per `intent_id` with status `leased`.
3. **Scheduler Ordering** ‚Äì The deterministic scoring function `Score(I)` together with the tie‚Äëbreaker `hash(intent_id)` defines a total order over the set of queued intents.
4. **IO Purity** ‚Äì All side‚Äëeffects during execution are confined to the **IO Fence Registry** (read‚Äëonly snapshot, queued writes). No live fetches or uncontrolled async I/O are permitted.
5. **Causal DAG Consistency** ‚Äì The Syscall DAG generated by the LangGraph layer must be a directed acyclic graph; cycles cause a hard failure.

## 2. Execution Context Virtual Machine (ECVM)

### 2.1 Data Model
```ts
export type ExecutionContext = {
  intent_id: string;
  ecvm_seed: string;                // cryptographic seed for RNG & clock
  logical_clock: number;            // monotonically increasing step counter
  scheduler_epoch: string;          // weight‚Äëset identifier (v7.4 epoch)
  io_mode: 'live' | 'shadow' | 'replay';
  now(): number;                    // frozen clock
  random(): number;                 // deterministic RNG
  llm_policy_snapshot: string;      // hash of LLM routing policy state
  event_sequence: number;           // global async ordering token
};
```

### 2.2 Core Primitives
* **FrozenClock** ‚Äì `now(step) = base + step`, where `base = hash(seed)`.
* **SeededRNG** ‚Äì `next() = hash(seed + counter)` yielding a float in `[0,1)`.
* **IO Fence Registry** ‚Äì declares allowed operations (`dbReadSnapshot`, `queueWrite`). Any attempt to call disallowed APIs throws `IOFenceViolation`.

## 3. Symbolic Execution Constraints (v7.3)

The **Symbolic Simulator** operates only on the **LangGraph decision nodes**, **Syscall DAG edges**, and **Scheduler transitions**. It builds a **PathSet** of possible execution branches under the current context constraints. The **Path Collapse Constraint** requires:
```
|PathSet| === 1
```
Violation raises a **NonDeterministicViolation** error and aborts the intent.

## 4. Proof‚Äëof‚ÄëExecution Certificate (PoE)

Every successful intent produces a JSON certificate stored in `kernel_execution_certificates`:
```json
{
  "intent_id": "uuid",
  "execution_hash": "hex256",
  "state_diff_hash": "hex256",
  "scheduler_trace_hash": "hex256",
  "io_trace_hash": "hex256",
  "ecvm_seed": "string",
  "verification_result": "valid",
  "proof_signature": "base64(sig)"
}
```
The certificate is signed with the kernel‚Äôs master private key; verification uses the public key baked into the runtime.

## 5. Scheduler Deterministic Scoring (v7.2.1)

```
Score(I) = w_urgency¬∑urgency + w_priority¬∑priority + w_efficiency¬∑efficiency - w_cost¬∑cost
```
Weights are loaded *once per epoch* via `kernel_scheduling_epochs`. The tie‚Äëbreaker is `hash(intent_id)`. The ordering is a total order ‚Äì no randomness.

## 6. Offline Optimization Layer (v7.4)

* **Learning Engine** reads immutable execution traces, drift heatmaps, and failure metrics.
* It proposes a new weight set `W_new`. A simulation run validates `determinism_score == 1.0`. If passed, a new epoch row is inserted and becomes active for future boots.
* Runtime never queries DB per request for weights; it loads the epoch snapshot at startup.

## 7. Failure State Taxonomy

| State | Description |
|-------|-------------|
| `VALID` | All invariants hold, PoE generated. |
| `NON_DETERMINISTIC_VIOLATION` | Symbolic PathSet > 1. |
| `LEASE_CONFLICT` | Duplicate lease detected. |
| `IO_FENCE_VIOLATION` | Disallowed I/O attempted. |
| `SCHEDULER_INCONSISTENCY` | Order mismatch detected during replay. |
| `VERIFICATION_FAILURE` | Proof generation or signature verification failed. |
